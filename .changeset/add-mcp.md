Add `@stone-js/mcp`: the agent-native brain of Stone.js. A curated, machine-readable map of the framework (concepts, modules, best-practices, known gaps) exposed as (1) ready-to-serve MCP tools (`stoneMcpTools` — plug into `@stone-js/mcp-adapter`) so a coding agent can query the framework in real time instead of scanning packages, (2) `generateLlmsTxt()` / `generateLlmsFullTxt()` for the `llms.txt` standard, and (3) `createReportTools()` to open bug/feature GitHub issues from the dev loop. "The LLM masters the context, you master the domain."
